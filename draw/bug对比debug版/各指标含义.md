## 整体流程概览

算法分为四个阶段：社区初始化 → 社区调整 → 信息提取（加噪） → 图重建。隐私预算 ε 被分成三份（e1、e2、e3），分别用在前三个阶段，保证整体满足 ε-差分隐私。

## 一、数据准备与原始图特征计算

```python
mat0, mid = get_mat(data_path)
mat0_graph = nx.from_numpy_array(mat0, create_using=nx.Graph)
```

从文件读入边列表，转成邻接矩阵 `mat0`，再构建 NetworkX 图对象。`mid` 是节点 ID 映射。

接下来计算原始图的一系列**基准指标**，后面用来和合成图对比：

- `mat0_par`：Louvain 社区划分
- `mat0_degree` / `mat0_deg_dist`：度序列和度分布
- `mat0_evc`：特征向量中心性（衡量节点重要性）
- `mat0_diam`：直径（最长最短路径）
- `mat0_cc`：全局聚类系数（传递性）
- `mat0_mod`：模块度

## 二、隐私预算分配

```python
e1 = e1_r * epsilon      # 社区初始化
e2 = e2_r * epsilon      # 社区调整
e3 = (1 - e1_r - e2_r) * epsilon  # 信息提取
```

默认各占 1/3。e3 再同时用于边向量（`ed`）和度序列（`ev`）的噪声添加。Laplace 噪声的尺度参数分别为 `1/ed` 和 `2/ev`（敏感度不同，边向量敏感度为1，度序列敏感度为2）。

## 三、社区初始化与调整

```python
mat1_pvarr1 = community_init(mat0, mat0_graph, epsilon=e1, nr=N, t=t)
mat1_par1 = comm.best_partition(mat0_graph, part1, epsilon_EM=e2)
```

第一步用隐私预算 e1 做一个初始的社区划分（`community_init` 内部细节在 `utils` 中），`N` 是采样节点数，`t` 是分辨率参数。第二步用 e2 进一步通过指数机制（EM）优化社区划分。最终 `mat1_pvarr` 是每个节点的社区标签，`mat1_pvs` 是每个社区包含的节点列表。

## 四、信息提取（加噪）—— 这是图重建的数据来源

### 4.1 边向量（Edge Vector）

```python
for i in range(comm_n):
    pi = mat1_pvs[i]
    ev_mat[i,i] = np.sum(mat0[np.ix_(pi,pi)])  # 社区i内部边数×2
    for j in range(i+1, comm_n):
        pj = mat1_pvs[j]
        ev_mat[i,j] = int(np.sum(mat0[np.ix_(pi,pj)]))  # 社区i和j之间的边数
```

`ev_mat` 是一个 **社区级别的边数矩阵**：对角线是社区内部边数，非对角线是社区间边数。这把一个 n×n 的邻接矩阵压缩成了 k×k 的小矩阵（k 为社区数），大幅降低了需要保护的信息量。

然后取上三角展平，加 Laplace 噪声，再后处理：

```python
ga = get_uptri_arr(ev_mat, ind=1)          # 取上三角（含对角线）
ga_noise = ga + laplace(0, ev_lambda, len(ga))  # 加噪
ga_noise_pp = FO_pp(ga_noise)              # 后处理（如取整、截断为非负）
ev_mat = get_upmat(ga_noise_pp, comm_n, ind=1)  # 还原为矩阵
```

### 4.2 度序列（Degree Sequence）

```python
for i in range(comm_n):
    dd1 = mat0[np.ix_(mat1_pvs[i], mat1_pvs[i])]
    dd1 = np.sum(dd1, 1)   # 社区内每个节点的"社区内度"
    dd1 = (dd1 + laplace(0, dd_lam, len(dd1))).astype(int)  # 加噪
    dd1 = FO_pp(dd1)       # 后处理
    dd1[dd1 < 0] = 0
    dd1[dd1 >= len(dd1)] = len(dd1) - 1  # 截断：度不能超过社区大小-1
```

注意这里只提取了**社区内度**（一个节点连接同社区其他节点的边数），而不是全局度。每个社区单独加噪，噪声量与社区无关（全局统一的 `dd_lam`）。

## 五、图重建（核心部分）⭐

这是整个算法最关键的部分——根据加噪后的边向量和度序列，生成合成图。

### 5.1 社区内部边的生成

```python
for i in range(comm_n):
    dd_ind = mat1_pvs[i]      # 该社区的节点索引
    dd1 = dd_s[i]             # 该社区加噪后的度序列
    mat2[np.ix_(dd_ind, dd_ind)] = generate_intra_edge(dd1)
```

`generate_intra_edge(dd1)` 根据给定的度序列生成一个子图的邻接矩阵。这通常使用**配置模型（Configuration Model）**或类似方法：把每个节点的度看作"存根"（stub），随机配对存根来形成边。目标是生成的子图的度序列尽量匹配 `dd1`。

### 5.2 社区间边的生成

```python
for j in range(i+1, comm_n):
    ev1 = ev_mat[i,j]        # 社区i和社区j之间应有的边数
    pj = mat1_pvs[j]
    if ev1 > 0:
        c1 = np.random.choice(pi, ev1)  # 从社区i中随机选ev1个节点
        c2 = np.random.choice(pj, ev1)  # 从社区j中随机选ev1个节点
        for ind in range(ev1):
            mat2[c1[ind], c2[ind]] = 1   # 连边
            mat2[c2[ind], c1[ind]] = 1
```

社区间边的生成很简单粗暴：**均匀随机**。已知两个社区之间需要 `ev1` 条边，就从两个社区中各随机抽 `ev1` 个节点配对连边。这里没有考虑度偏好，是完全随机的。

⚠️ 注意这里有个 **bug**：循环变量用的是 `pi`，但 `pi` 是在外层 `i` 循环的 4.1 步中赋值的边向量代码里定义的，在图重建这段代码里其实应该是 `dd_ind`（即 `mat1_pvs[i]`）。代码能跑通是因为 Python 变量作用域的特性——`pi` 恰好保留了上一次迭代的值，但逻辑上应当用 `mat1_pvs[i]`。

### 5.3 矩阵对称化与去重

```python
mat2 = mat2 + np.transpose(mat2)
mat2 = np.triu(mat2, 1)        # 只保留严格上三角
mat2 = mat2 + np.transpose(mat2)  # 再对称化
mat2[mat2 > 0] = 1              # 二值化（去除重边）
```

因为社区内的 `generate_intra_edge` 可能返回上三角矩阵，社区间连边也可能造成重复，所以这一步做对称化 + 去重，确保最终得到一个合法的无向简单图邻接矩阵。

## 总结

## 1. NMI（归一化互信息）

NMI 衡量两个**划分**之间的相似程度，来源于信息论。

假设原始图社区划分为 A = {A₁, A₂, ...}，合成图社区划分为 B = {B₁, B₂, ...}，总共 n 个节点。先构造一个**列联表** H，其中 Hᵢⱼ 表示同时属于 Aᵢ 和 Bⱼ 的节点数。

$$NMI(A,B) = \frac{-2 \sum_{i}\sum_{j} H_{ij} \log\frac{H_{ij} \cdot n}{x_i \cdot y_j}}{\sum_{i} x_i \log\frac{x_i}{n} + \sum_{j} y_j \log\frac{y_j}{n}}$$

其中 xᵢ = 第 i 行之和（Aᵢ 的大小），yⱼ = 第 j 列之和（Bⱼ 的大小）。

**举例：** 6 个节点，原始划分 A = {1,2,3}, {4,5,6}，合成图划分 B = {1,2,3}, {4,5,6}。

列联表：

| | B₁ | B₂ |
|---|---|---|
| A₁ | 3 | 0 |
| A₂ | 0 | 3 |

两个划分完全一致，NMI = 1.0。

如果合成图划分变成 B = {1,2,4}, {3,5,6}：

| | B₁ | B₂ |
|---|---|---|
| A₁ | 2 | 1 |
| A₂ | 1 | 2 |

此时 NMI 大约是 0.13，说明两个划分差异较大。

**直觉理解：** 如果知道了 A 的划分，能帮你多大程度"猜对" B 的划分？互信息越大，两个划分越一致。归一化后值域为 [0,1]。

## 2. EVC Overlap（特征向量中心性 Top-1% 节点重叠率）

**特征向量中心性（EVC）** 的含义是：一个节点的重要性取决于它邻居的重要性。数学上，它是邻接矩阵最大特征值对应的特征向量的各分量。

计算步骤：
1. 对原始图和合成图分别计算所有节点的 EVC 分数
2. 各取 EVC 排名前 1% 的节点集合 N 和 N̂
3. 计算重叠率：

$$Overlap = \frac{|N \cap \hat{N}|}{|N|}$$

**举例：** 100 个节点的图，取前 1% 就是 top-1 个节点。

原始图中 EVC 最高的节点是节点 42，合成图中 EVC 最高的节点也是 42，则 Overlap = 1/1 = 1.0。如果合成图最高的是节点 78，则 Overlap = 0/1 = 0。

再比如 1000 个节点的图，取前 1% 即前 10 个节点。原始图 top-10 是 {3,7,12,15,20,25,30,35,40,45}，合成图 top-10 是 {3,7,12,15,20,50,60,70,80,90}。交集是 {3,7,12,15,20}，共 5 个。Overlap = 5/10 = 0.5。

---

## 3. EVC MAE（特征向量中心性均绝对误差）

在上一步中，我们已经得到原始图 top-1% 节点的 EVC 分数排名。现在比较**这些节点**在原始图和合成图中 EVC 分数的数值差异。

$$MAE_{EVC} = \frac{1}{T}\sum_{i=1}^{T}|s_i - \hat{s}_i|$$

其中 T 是 top-1% 的节点数，sᵢ 是原始图中第 i 个最重要节点的 EVC 分数，ŝᵢ 是合成图中第 i 个最重要节点的 EVC 分数。

**举例：** 假设 top-3 节点的 EVC 分数：

| 排名 | 原始图 sᵢ | 合成图 ŝᵢ | 差值 |
|---|---|---|---|
| 1 | 0.45 | 0.42 | 0.03 |
| 2 | 0.38 | 0.35 | 0.03 |
| 3 | 0.30 | 0.20 | 0.10 |

MAE = (0.03 + 0.03 + 0.10) / 3 = 0.053

注意这里比较的是"排名对齐"后的分数差，即原始图排第 1 的 vs 合成图排第 1 的，而不是同一个节点在两张图里的分数差。

## 4. Degree KL（度分布 KL 散度）

先分别统计两张图的**度分布**（每个度值出现的概率），然后计算 KL 散度：

$$D_{KL}(P \| \hat{P}) = \sum_{x} P(x) \log\frac{P(x)}{\hat{P}(x)}$$

其中 P(x) 是原始图中度为 x 的节点占比，P̂(x) 是合成图中度为 x 的节点占比。

**举例：** 5 个节点的图。

原始图度序列：[2, 2, 3, 3, 4]

度分布 P：P(2)=2/5=0.4, P(3)=2/5=0.4, P(4)=1/5=0.2

合成图度序列：[2, 2, 2, 3, 3]

度分布 P̂：P̂(2)=3/5=0.6, P̂(3)=2/5=0.4, P̂(4)=0

这里 P̂(4)=0 会导致 log 无穷大，实际计算中通常会做平滑处理（比如加一个很小的常数）。假设平滑后 P̂(4) = 0.001：

$$D_{KL} = 0.4 \log\frac{0.4}{0.6} + 0.4\log\frac{0.4}{0.4} + 0.2\log\frac{0.2}{0.001}$$
$$= 0.4 \times (-0.405) + 0 + 0.2 \times 5.298 = -0.162 + 0 + 1.06 = 0.898$$

如果两个度分布完全相同，KL = 0。差异越大，KL 越大。

**直觉理解：** KL 散度衡量的是"用合成图的度分布去近似原始图的度分布，损失了多少信息"。

## 5. Diameter RE（直径相对误差）

**直径**是图中所有最短路径中最长的那个。如果图不连通，通常取最大连通分量的直径。

$$RE_{Diam} = \frac{|\hat{D} - D|}{\max(\delta, D)}$$

其中 D 是原始图直径，D̂ 是合成图直径，δ 是一个很小的常数防止分母为零。

**举例：**

原始图直径 D = 8（从一端到另一端最远要经过 8 条边），合成图直径 D̂ = 10。

$$RE = \frac{|10-8|}{8} = 0.25$$

意思是合成图的"最远距离"比原始图偏大了 25%。

如果合成图直径也是 8，RE = 0，完美还原。

---

## 6. CC RE（聚类系数相对误差）

**全局聚类系数（传递性）** 的定义是：

$$CC = \frac{3 \times \text{三角形个数}}{\text{连通三元组个数}}$$

"连通三元组"指三个节点中有至少两条边相连的组合（即一个中心节点连接另外两个），"三角形"指三个节点两两相连。

$$RE_{CC} = \frac{|\hat{Y} - Y|}{\max(\delta, Y)}$$

**举例：**

考虑一个 4 节点图：1-2, 2-3, 3-1, 3-4

连通三元组（以中心节点列举）：(1,2,3)以2为中心, (1,2,3)以1为中心, (1,2,3)以3为中心, (2,3,4)以3为中心 → 共 4 个

三角形：{1,2,3} 这一个 → 3 × 1 = 3

原始 CC = 3 / 4 = 0.75

假设合成图的 CC = 0.5：

$$RE = \frac{|0.5 - 0.75|}{0.75} = 0.333$$

**直觉理解：** 聚类系数反映"你朋友的朋友也是你朋友"的程度。社交网络通常有较高的聚类系数。RE 衡量合成图在这方面的偏差。

---

## 7. Modularity RE（模块度相对误差）

**模块度 Q** 衡量社区划分的质量：

$$Q = \sum_C \left[\frac{\Sigma_{in}}{2m} - \left(\frac{\Sigma_{tot}}{2m}\right)^2\right]$$

其中对每个社区 C：Σᵢₙ 是社区内部边权之和，Σₜₒₜ 是社区中节点所有边权之和，m 是全图边权总和。

$$RE_{Mod} = \frac{|\hat{Q} - Q|}{\max(\delta, Q)}$$

**举例：**

一个 6 节点图分成两个社区 {1,2,3} 和 {4,5,6}。社区内部边很密，社区之间只有 1 条边。

假设 m = 10（总共 10 条边）。社区 1 内部有 3 条边（Σᵢₙ=6，因为每条边在邻接矩阵里出现两次），Σₜₒₜ = 8（社区 1 节点的总度数）。

社区 1 的贡献：6/20 - (8/20)² = 0.3 - 0.16 = 0.14

两个社区类似，假设总 Q = 0.28。

如果合成图的 Q̂ = 0.20：

$$RE = \frac{|0.20 - 0.28|}{0.28} = 0.286$$

**直觉理解：** 模块度高意味着社区内部连接紧密、社区之间连接稀疏。Q 接近 1 说明社区结构非常明显，接近 0 说明和随机图差不多。RE 衡量合成图对这种宏观社区结构特征的保持程度。

## 总览

| 指标 | 衡量维度 | 公式核心 | 方向 |
|---|---|---|---|
| NMI | 社区结构 | 信息论互信息 | 越高越好 |
| Overlap | 关键节点 | Top-1%集合交集比 | 越高越好 |
| MAE | 关键节点 | EVC分数绝对差均值 | 越低越好 |
| KL | 度分布 | KL散度 | 越低越好 |
| Diam RE | 连通性 | 直径相对误差 | 越低越好 |
| CC RE | 局部拓扑 | 聚类系数相对误差 | 越低越好 |
| Mod RE | 宏观拓扑 | 模块度相对误差 | 越低越好 |
